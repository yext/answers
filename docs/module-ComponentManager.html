Class: module:ComponentManager

module:ComponentManager()

ComponentManager is a Singletone that contains both an internal registry of eligible components to be created, as well as keeps track of the current instantiated and active components. ALL components should be constructed using the {ComponentManager} via its `create` method.

_core :Core

A local reference to the core library dependency The Core contains both the storage AND services that are needed for performing operations like search and auto complete. The storage is the source of truth for the state of ALL components. Whenever the storage is updated, the state gets pushed down to the necessary components.
